# AI Craft Hub (MVP)

Веб-платформа для ремесленников и малого бизнеса: маркетплейс, AI-ассистент (OpenAI), заявки на микрокредит с AI-оценкой, eco / ESG-метки и фильтры.

## Структура

```
project/
├── app.py           # Flask, маршруты, сиды
├── models.py        # User, Product, Order, CreditApplication, избранное
├── ai.py            # OpenAI + демо-режим без ключа
├── credit.py        # Оценка заявки (вызывает ai.py)
├── requirements.txt
├── templates/
├── static/
└── database.db      # создаётся при первом запуске (не в git)
```

## Как запустить

1. **Python 3.10+**

2. Виртуальное окружение и зависимости:

   ```powershell
   cd project
   python -m venv venv
   .\venv\Scripts\pip install -r requirements.txt
   ```

3. (Необязательно) файл `.env` в папке `project`:

   - Скопируйте `.env.example` в `.env`.
   - Укажите `OPENAI_API_KEY` для реальных ответов AI.
   - Без ключа приложение работает в **демо-режиме** (заглушки и эвристика для кредита).

4. Запуск:

   ```powershell
   .\venv\Scripts\python app.py
   ```

   Откройте в браузере: `http://127.0.0.1:5000`

При первом старте создаётся SQLite `database.db` и подставляются **тестовые данные** (8 товаров, 2 демо-пользователя).

## Примеры использования

1. **Каталог и eco-фильтр**  
   Меню «Каталог» → поиск, категория (`eco`, `handmade`, `agro`, `bio`), чекбокс «Только eco-товары».

2. **Чат с AI**  
   Войдите → «AI-чат». Спросите, например: «Как повысить продажи handmade на маркетплейсе?»  
   С ключом OpenAI ответы полноценные; без ключа — короткие демо-ответы.

3. **Генерация описания товара**  
   Войдите как продавец → «Новый товар» → кнопка «AI: сгенерировать описание».

4. **Eco-рекомендации**  
   Откройте карточку товара → «AI: рекомендации по экологичности».

5. **AI Микрокредит (финтех MVP)**  
   `/credit` — описание сервиса, преимущества, **симулятор ежемесячного платежа** (аннуитет, 21% годовых, демо), форма: имя, сумма, доход, продажи за 30 дней, стаж (мес.), категория (`eco`, `handmade`, `agro`, `bio`), описание бизнеса → **Подать заявку**.  
   Скоринг (`credit.py`): больше продаж и доход → выше шанс; **eco/bio и ESG в тексте** — бонус и бейдж «устойчивый бизнес»; стаж **> 6 мес.** — бонус; тяжёлая долговая нагрузка (сумма к доходу) — штраф.  
   OpenAI (`ai.narrate_credit_assessment`) объясняет результат и даёт советы (как повысить шанс, как развивать бизнес); без ключа — демо-тексты.  
   Редирект на `/credit_result?application_id=…`: риск (цвет), вероятность %, рекомендуемая сумма, рекомендация **одобрить / доработать**, статус **pending / approved / rejected**. История: `/credit_history`. Демо-заявки в сиде для `wood@demo.craft`, `clay@demo.craft`, `buyer@craft.demo`.

6. **Профиль (личный кабинет)**  
   После входа: «Профиль» (`/profile`) — карточка пользователя, для продавца: таблица товаров (добавить / редактировать / удалить), блок статистики (демо-цифры), доля eco-товаров, AI-рекомендации (кнопка отправляет `POST /ai_advice`), ссылка на заявку микрокредита и список заявок.

7. **Добавление товара**  
   Продавец: «Новый товар» или `/add_product` — форма с категориями `eco`, `handmade`, `agro`, `bio`, чекбокс eco/ESG, кнопки AI (описание по названию, распознавание по фото). Старый путь `/product/create` ведёт на тот же обработчик.

8. **Оформление заказа (checkout, MVP)**  
   С карточки товара или витрины: **Купить** → `/checkout/<id>` — товар, продавец, количество, итог; форма: имя, телефон, адрес; **Оформить заказ** создаёт запись `Order` (сначала `pending`, затем сразу `completed` — имитация успешной оплаты) и редирект на `/order_success?order_id=…` с сообщением об успехе, похожими товарами и блоком **eco-альтернатив**. Кнопка **«Спросить AI перед покупкой»** вызывает `POST /api/ai/purchase-advice` (JSON: стоит ли покупать, eco, плюсы/минусы). Демо-заказы создаются сидом для `buyer@craft.demo`.

9. **Главная и каталог товаров**  
   `/` и `/catalog` — одна логика витрины: поиск по названию и описанию, фильтры (категория, цена от/до, только eco), сортировка (новизна, цена, демо-популярность), доля eco на текущей выкладке. Блок **«Подобрать товар с помощью AI»** (`POST /ai_recommend`) сохраняет порядок в сессии; откройте страницу с `?ai=1` или снова нажмите «Подобрать» после редиректа. Избранные товары: `POST /favorite` (`product_id`, `action`), список — `/favorites` (нужен вход). Карточки: «Подробнее», «Купить» (демо), ссылка на продавца.

10. **Каталог продавцов**  
   Меню «Продавцы» (`/sellers`) — карточки мастерских с демо-рейтингом, числом товаров (с фото), долей eco и бейджем **Eco seller** (если ≥35% eco-товаров). Поиск по словам в имени, описании и названиях лотов; фильтры по категории товаров, eco, минимальному рейтингу и числу позиций.  
   Форма **«Найти продавца с помощью AI»** (`POST /ai_seller_search`) ранжирует витрины под ваш запрос; результат открывается как `/sellers?ai=1` (кнопка «Сбросить подбор AI» очищает порядок).  
   Войдите как покупатель — **избранное** (`POST /favorite_seller`, поля `seller_id`, `action`, опционально `next`) и список избранных на странице каталога.  
   Страница продавца (`/seller/<id>`, шаблон `seller_profile.html`): категории, eco-%, демо-статистика, кнопка **«Проанализировать продавца»** (`POST /seller/<id>/analyze`) — блок с сильными/слабыми сторонами и потенциалом роста (OpenAI или демо).

## Маршруты профиля и товаров

| Метод | Путь | Назначение |
|--------|------|------------|
| GET | `/profile` | Личный кабинет |
| GET/POST | `/add_product` | Новый товар (и `/product/create`) |
| GET/POST | `/product/<id>/edit` | Редактирование |
| POST | `/delete_product` | Удаление (поле `product_id`) |
| POST | `/ai_advice` | Сгенерировать блок советов AI для профиля |

## Маршруты микрокредита

| Метод | Путь | Назначение |
|--------|------|------------|
| GET/POST | `/credit` | Страница заявки и симулятор платежа |
| GET | `/credit_result?application_id=` | Результат оценки и AI-тексты |
| GET | `/credit_history` | Список заявок пользователя |

## Маршруты витрины товаров

| Метод | Путь | Назначение |
|--------|------|------------|

| POST | `/ai_recommend` | AI-подбор (`query`, `source`: index \| catalog) |
| POST | `/favorite` | Избранное по товару (`product_id`, `action`, `next`) |
| GET | `/favorites` | Список избранных товаров |
| GET | `/product/<id>` | Карточка товара, похожие, eco-блок |

## Маршруты checkout

| Метод | Путь | Назначение |
|--------|------|------------|
| GET/POST | `/checkout/<product_id>` | Страница покупки и создание заказа |
| GET | `/order_success?order_id=` | Успех, похожие и eco-альтернативы |
| POST | `/api/ai/purchase-advice` | JSON `product_id` → совет перед покупкой |

## Маршруты каталога продавцов

| Метод | Путь | Назначение |
|--------|------|------------|
| GET | `/sellers` | Список, поиск, фильтры, порядок после AI (`?ai=1`), `?clear_ai=1` — сброс подбора |
| POST | `/ai_seller_search` | Текстовый запрос → ранжирование продавцов (сессия + редирект на `?ai=1`) |
| GET | `/seller/<id>` | Публичный профиль и товары |
| POST | `/seller/<id>/analyze` | AI-анализ витрины (нужен вход) |
| POST | `/favorite_seller` | Избранное: `seller_id`, `action` = add / remove / toggle |

## API для форм (JSON)

- `POST /api/ai/describe` — тело: `{"title":"...", "category":"handmade", "hints":"..."}`  
- `POST /api/ai/eco` — `{"product_id": 1}` или `{"title":"...", "description":"..."}`  

Требуется сессия (пользователь должен быть залогинен).

## Локализация (ru / kk / en)

- Строки страниц и flash-сообщений расширения: `project/locales/pages_<lang>.json` (подмешиваются в `i18n._load_file`).

## Безопасность (MVP → прод)

- **CSRF:** `Flask-WTF` защищает POST-формы; в шаблонах — скрытое поле `csrf_token`, для `fetch` — заголовок `X-CSRFToken` (см. `base.html` и `window.csrfHeaders`).
- **Сессия:** `SESSION_COOKIE_HTTPONLY`, `SameSite=Lax`; **Secure** включается при `FLASK_ENV=production` или `CRAFT_HUB_PRODUCTION=1`.
- **Лимиты:** на `login` и `register` — `Flask-Limiter` (отключается при `TESTING`).
- Секреты только в `.env`, не коммитьте реальный ключ.

## Фоновые задачи и AI

- Тяжёлые вызовы OpenAI выполняются синхронно в запросе; для высокой нагрузки обычно выносят в очередь (Celery / RQ и т.д.) — в MVP не реализовано.

## Важно

- MVP не является финансовым продуктом: оценка кредита — демонстрация.
- Для продакшена задайте надёжный `FLASK_SECRET_KEY` в `.env` и включите HTTPS + переменные окружения из `.env.example`.
